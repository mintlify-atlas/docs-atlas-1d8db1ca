The `RestClient` class is the primary entry point for all REST API interactions with Gate.io. It covers every product area the exchange offers — from spot and margin trading to futures, delivery, options, wallet management, sub-accounts, and beyond. Every method returns a `Promise`, making it fully compatible with `async/await` patterns in both Node.js and browser environments.

## Installation and Import

```typescript
import { RestClient } from 'gateio-api';
```

## Constructor

```typescript
new RestClient(options?: RestClientOptions, requestOptions?: AxiosRequestConfig)
```

Both parameters are optional. If you only need public (unauthenticated) endpoints, you can instantiate with no arguments at all.

<ParamField path="options" type="RestClientOptions" optional>
  Configuration object for the REST client. All fields are optional.

  <Expandable title="RestClientOptions fields">
    <ParamField path="apiKey" type="string" optional>
      Your Gate.io API key. Required for authenticated endpoints.
    </ParamField>
    <ParamField path="apiSecret" type="string" optional>
      Your Gate.io API secret. Required for authenticated endpoints.
    </ParamField>
    <ParamField path="recvWindow" type="number" optional>
      Max size of the request window in milliseconds for signed API calls. Overrides the global default.
    </ParamField>
    <ParamField path="strictParamValidation" type="boolean" optional>
      Default: `false`. When `true`, throws errors if any request parameters are `undefined`.
    </ParamField>
    <ParamField path="baseUrl" type="string" optional>
      Optionally override the API protocol and domain. Example: `'https://api.gate.io'`.
    </ParamField>
    <ParamField path="baseUrlKey" type="GateBaseUrlKey" optional>
      Manually select one of the pre-configured base URLs in the library (`'live'`, `'futuresLiveAlternative'`, or `'futuresTestnet'`).
    </ParamField>
    <ParamField path="parseExceptions" type="boolean" optional>
      Default: `true`. Whether to post-process and re-throw request exceptions from the HTTP layer.
    </ParamField>
    <ParamField path="keepAlive" type="boolean" optional>
      Enable HTTP KeepAlive for REST API requests via axios.
    </ParamField>
    <ParamField path="keepAliveMsecs" type="number" optional>
      How often to send TCP KeepAlive packets when `keepAlive` is `true`. Default: `1000` ms.
    </ParamField>
    <ParamField path="customSignMessageFn" type="(message: string, secret: string) => Promise<string>" optional>
      Provide a custom HMAC signing function. Useful for using Node.js's faster `createHmac` instead of the Web Crypto API. See the examples folder for a demonstration.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField path="requestOptions" type="AxiosRequestConfig" optional>
  Low-level Axios request configuration passed directly to the HTTP client. Use this for proxy settings, custom headers, or timeout overrides.
</ParamField>

## Instantiation Examples

**Public endpoints only (no authentication):**

```typescript
import { RestClient } from 'gateio-api';

const client = new RestClient();

const tickers = await client.getSpotTicker({ currency_pair: 'BTC_USDT' });
console.log(tickers);
```

**Authenticated client:**

```typescript
import { RestClient } from 'gateio-api';

const client = new RestClient({
  apiKey: 'YOUR_API_KEY',
  apiSecret: 'YOUR_API_SECRET',
});

const accounts = await client.getSpotAccounts();
console.log(accounts);
```

**Futures testnet:**

```typescript
import { RestClient } from 'gateio-api';

const client = new RestClient({
  apiKey: 'YOUR_TESTNET_API_KEY',
  apiSecret: 'YOUR_TESTNET_API_SECRET',
  baseUrlKey: 'futuresTestnet',
});
```

## Authentication

Authentication is handled automatically for any method that requires it. When `apiKey` and `apiSecret` are present in the constructor options, the client signs every request that touches a private endpoint. You do not need to call any separate login method.

Methods that require authentication are marked with a lock icon (🔒) in the [endpoint function list](https://github.com/tiagosiebler/gateio-api/blob/master/docs/endpointFunctionList.md) on GitHub.

## TypeScript Support

All request parameter types and response shapes are exported from the package. Import them alongside the client:

```typescript
import {
  RestClient,
  RestClientOptions,
} from 'gateio-api';

// Request and response types are available from the package exports
import type { SubmitSpotOrderReq, SpotOrder } from 'gateio-api';
```

The SDK is written entirely in TypeScript, so your IDE will provide full autocompletion and inline documentation for every method parameter and return type.
